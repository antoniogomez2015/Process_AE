	Workflow:
	_________




	Volcado a un sistema tipo Duik:
	_______________________________



	Características:
	________________

	_una vez hemos realizado el proceso de rigging, sea un sistema unico ó de un sistema compuesto de varios

	sistemas en una sólo en la conformación de un character set


	_en ambos casos tendremos la posibilidad de hacer un volcado del sistema nativo ha un sistema de layers,

	auxiliares de tipo null, convencional


	_las cuáles, al ser varios sistema integrados en uno sólo, todos los sistemas quedarán integrados ó 

	emparentados en si como un character rig, terminado


	_de ésta manera, obtendremos un sistema de rig, convencional automático con la ventaja de que éste 

	sistema contará con las ventajas del sistema nativo en las articulaciones


	_es decir, que las articulaciones podrán ser rígidas ó articuladas u orgánicas en relación a nuestras

	necesidades


	_ya que la layer, principal "Torso/Chest", la cuál, contiene todos los efectos del sistema base nativo 

	y de todos los sistemas relacionados 


	_ésta layer, siempre mantendrá todos éstos efectos intactos posteriormente de haber realizado el proceso 

	de conversión ó volcado al sistema convencional de nulls

	
	_es decir, éstos efectos no se perderán ó dejarán de funcionar a la hora de realizar el volcado al 

	sistema de layers, auxiliares de tipo null


	_de ésta manera, podremos hacer ajustes posteriormente de hacer realizado el sistema de conversión




	Volcado:
	________

	_de ésta manera, una vez hemos realizado todo el sistema nativo para la generación de rig, de todas las

	partes de un character


	_podremos entonces comenzar ha realizar el volcado de todas éstas partes ó rigs, relacionados para lo

	cuál, debemos de seleccionar la layer, principal que contiene a todos los efectos "Body/Torso/Chest"


	_y con ésta layers, seleccionada debemos ir al panel "effects control", shortcut, "F3", para que así,

	podamos acceder al efecto de cada parte del rig, del character


	_de ésta manera, empezaremos el rig, principal del "Chest", para lo cuál, debemos ir a las opciones de:



		_"Effects Controls/Chest/Quick Bone Set"


	_y allí, seleccionar la opción de: 



		_"Set With Controllers"


	_para que así, sistema elimine al sistema nativo de rig, de esa parte del character, y cree a su vez, uno 

	ó dos layers, de tipo null, dependiendo del rig, respectivamente


	_éstas nuevas layers, el sistema las colocará en la parte superior de la composición respectiva 


	_éstas layers, el sistema las renombrará con el mismo nombre de la layer, base a la cuál, hacen referencia

	más al principio de cada nombre el sistema les agregará un prefijo el cuál, es un sigla de "Control"


		_"C_"


	_de ésta manera, podremos entrar en el efecto controlador de cada parte del rig, del character, dentro de

	la layer, base "Body/Torso/Chest"


	_y proceder ha repetir el proceso de conversión yendo ha _"Effects Controls/Chest/Quick Bone Set", y 

	seleccionando la opción de "Set With Controllers"


	_de ésta manera, habremos creado automáticamente un sistema de rig, completo para su uso de forma 

	convencional


	_lo que nos permitirá integrar éste sistema con otros tipos de sistemas más regulares así, como también, 

	con otros plugins, y efectos nativos, otros




	Handles:
	________

	_y para un mejor manejo y más práctico a la hora de animar el character, podremos crear controles más

	visibles y manejables que los por defecto a través, de null 


	_podremos así, hacer uso en éste punto de las opciones del plugin Duik, ó de formas personalizadas 

	realiadas por nosotros para emparentar así, handles, personalizados


	_de ésta manera, procederemos ha seleccionar la layer, de tipo null, generada por el procedo de volcado

	y con el nombre de la articulación a la cuál, deseamos crearle un handle


	_para a continuación ir a las opciones del plugings, "Duik/Rigging/Controllers", y allí, indicar

	un tamaño y un color para el handle, ha crear


	_de ésta manera, podremos repetir éstos pasos hasta crear un handle, para cada uno de los null, de 

	control generado por el proceso de volcado


	_para por último proceder con el proceso de emparentamiento respectivo, de ésta manera, por ejemplo,

	proceder ha emparentar el null, ej: "L_Arm", con el handle, que hemos creado en el pluging, Duik,

	con el mismo nombre "C_L_Arm"


	_de ésta manera, podremos repetir éstos pasos para el resto de controles y handles, respectivamente,

	para que así, podamos ahora controlar el character, de una manera más fácil y más visual a la hora

	de realizar el proceso de animación
